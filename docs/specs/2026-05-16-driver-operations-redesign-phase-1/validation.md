## Automated tests

- `tests/namespaces/test_entity_namespace.py`, `test_episodic_namespace.py`, `test_community_namespace.py`, `test_saga_namespace.py` — save / get / delete via the new API; assert embeddings are generated by the namespace, not the ops.
- `tests/test_graphiti_mock.py` — **continues to pass unchanged**. Phase 1 is non-breaking; any failure here is a regression to fix before merge.
- `tests/driver/test_operations_neo4j.py` — per-ops ABC compliance for the Neo4j implementation; parametrized fixture so the same suite can be re-targeted per backend in follow-up PRs.
- `tests/driver/test_transaction.py` — `async with driver.transaction() as tx: ...` rolls back on exception, commits on clean exit; default impl behaves identically to `driver.session()`.
- `tests/test_data_model_method_forwarding.py` — `EntityNode.save` calls into the namespace; assertable via a spy or method-patch.

## Manual criteria

- API discoverability: in a notebook, `g.nodes.<tab>` shows the new subnamespaces and `g.nodes.entity.<tab>` shows the operations.
- The new files under `graphiti_core/namespaces/` and `driver/neo4j/operations/` read as small, focused modules — no single file over ~200 LOC.

## Risks & rollback

- **Failure modes**: subtle behavior drift between a data-model method and its namespace forward (parameter defaults differ, exception types differ); per-driver ops files explode in count and become hard to navigate; non-Neo4j backends bit-rot because Phase 1 only ports Neo4j; the new namespace classes shadow attributes on `Graphiti` that users were already importing.
- **Rollback**: revert the PR series in reverse order. The data-model methods are kept intact by design, so revert is clean and leaves no orphaned state.

## Open questions

- Per-backend migration cadence: one big PR or one PR per backend? Lean toward per-backend for review; the spec leaves this open.
- Are the saga and community namespaces stable enough to commit to this API, or should they be marked **experimental** in 0.30 docs? Decide before tagging.
- Where does `add_triplet` live in the new model — on `Graphiti` (current) or on a namespace? Defer: keep on `Graphiti` for now.
